## Integration Issues (Phase 9)

### CRITICAL: is_hidden column not enforced in queries

**Affects:** SAFE-02, SAFE-03
**Severity:** High — breaks moderation enforcement loop

Migration 030 adds `is_hidden` to `checkins` and `checkin_comments`. `ModerationService.autoHideContent()` correctly writes `is_hidden = true` when SafeSearch flags content. However, **no content-serving query filters on this column**:

- `FeedService.ts` — `getFriendsFeed()`, `getEventFeed()`, `getHappeningNow()` — no `is_hidden` filter
- `CheckinQueryService.ts` — `getActivityFeed()`, `getCheckins()` — no `is_hidden` filter
- `DiscoveryService.ts` — recommendation queries — no `is_hidden` filter

**Result:** Content flagged by SafeSearch or hidden by admin moderation continues to appear in every feed. Admin approve/remove actions have no visible effect.

**Fix:** Add `AND (c.is_hidden IS NOT TRUE)` to all checkin WHERE clauses and `AND (cm.is_hidden IS NOT TRUE)` to comment queries.

### LOW: Password reset deep link not wired

**Affects:** SAFE-05
**Severity:** Low — documented, intentional deferral to Phase 10

The email contains `soundcheck://reset-password?token=X` but `app_router.dart` has no route for this deep link. The request half works end-to-end. Full round-trip requires Phase 10 Universal Links work.
