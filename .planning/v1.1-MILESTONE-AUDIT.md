### New Integration Gaps (2)

**1. Wrapped detail "Learn about Pro" broken navigation (P2):**
- `WrappedDetailScreen` (wrapped_detail_screen.dart:58) catches the 403 from `requirePremium()` and shows an inline lock screen
- The "Learn about Pro" button uses `Navigator.of(context).pushNamed('/pro')` — this fails silently because the app uses GoRouter exclusively (no MaterialApp named routes registered)
- **Fix:** Change to `GoRouter.of(context).push('/pro')` and add `import 'package:go_router/go_router.dart';`
- **Impact:** Non-premium users who reach the Wrapped detail screen can't navigate to Pro signup from that screen. Alternative path (profile → Pro screen) still works.

**2. Wrapped friend overlap block filter (P3):**
- `WrappedService.getFriendOverlap` (WrappedService.ts:262-285) joins `user_followers` to find friends who attended the same events
- Does NOT filter against the `blocks` table — a blocked user who is still in `user_followers` could appear in results
- **Impact:** Premium-only feature; user would need to (a) follow someone, (b) block them, (c) not unfollow them, (d) have attended the same events in the same year
- **No requirement violation** — blocking requirement (SAFE-04) specifies "interactions cease in both directions" which applies to feeds/content, not personal analytics
- **Recommended fix:** Add `AND c2.user_id NOT IN (SELECT blocked_user_id FROM blocks WHERE blocker_user_id = $1 UNION SELECT blocker_user_id FROM blocks WHERE blocked_user_id = $1)` to the friend overlap query
