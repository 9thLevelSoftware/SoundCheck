## Human Verification Required

### 1. Android Deep Link — Physical Device Test

**Test:** Install debug APK on Android device. Send a test email with a link formatted as `soundcheck://reset-password?token=TESTTOKEN`. Tap the link from the email client.
**Expected:** The SoundCheck app opens directly to the ResetPasswordScreen with the token pre-populated (form state shown, not error state).
**Why human:** Android intent-filter resolution requires a real device or emulator with the app installed; cannot be verified statically.

### 2. iOS Deep Link — Physical Device / Simulator Test

**Test:** On iOS device or simulator, open Safari and navigate to or tap `soundcheck://reset-password?token=TESTTOKEN`.
**Expected:** The SoundCheck app opens to ResetPasswordScreen (form state).
**Why human:** CFBundleURLTypes registration is validated at runtime by the iOS URL scheme dispatcher; static analysis cannot confirm the scheme is properly registered.

### 3. Auth Guard — Authenticated User Arriving via Deep Link

**Test:** While logged in, tap a `soundcheck://reset-password?token=X` deep link.
**Expected:** The auth guard redirect rule `if (isAuthenticated && isOnAuthPage) return '/feed'` will redirect away from the reset screen. Verify whether this is the intended UX or whether authenticated users should be able to access the reset flow.
**Why human:** The current guard redirects any authenticated user away from `/reset-password`. If an authenticated user clicks a stale reset link, they silently land on the feed with no indication the link was used. This is a behavioral edge case that needs a product decision, not a code fix.

### 4. Password Reset End-to-End — Backend Email Delivery

**Test:** Enter a valid account email in the Forgot Password screen and submit. Check inbox for reset email within ~30 seconds.
**Expected:** Email received with a `soundcheck://reset-password?token=<valid-token>` link.
**Why human:** Requires a configured `RESEND_API_KEY` environment variable on the backend. SUMMARY notes this is a pre-existing pending setup item. Cannot verify email delivery without live infrastructure.
