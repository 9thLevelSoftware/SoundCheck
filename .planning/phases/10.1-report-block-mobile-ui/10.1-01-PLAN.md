Backend Report API (from RESEARCH.md):
- POST /reports (DioClient path: '/reports' -- no /api prefix)
- Body: { contentType: 'checkin'|'comment'|'photo'|'user', contentId: string, reason: 'spam'|'harassment'|'inappropriate'|'copyright'|'other', description?: string }
- Response: { success: true, data: report, message: "Report submitted..." }
- Rate limit: 10 reports/user/day (429 when exceeded)
- Deduplication: UNIQUE constraint -- same user can't report same content twice
- CRITICAL: For photo reports, use the CHECK-IN ID as contentId (photos are stored as image_urls on checkins table, not as separate entities)

**report_repository.dart:**
- Class `ReportRepository` with constructor taking `DioClient` (same pattern as `RsvpRepository`, `DiscoveryRepository`)
- Method `submitReport({required String contentType, required String contentId, required String reason, String? description})` that POSTs to `/reports`
- Path is `/reports` (NOT `/api/reports` -- DioClient baseUrl already includes `/api`)
- No return value needed -- success is indicated by no exception thrown

**report_providers.dart:**
- Manual Riverpod providers (NOT @riverpod codegen -- per Phase 10 decision [10-05])
- `reportRepositoryProvider`: `Provider<ReportRepository>` that watches `dioClientProvider`
- Import dioClientProvider from `../../../../core/providers/providers.dart`

**report_bottom_sheet.dart:**
- Create a `ReportBottomSheet` ConsumerStatefulWidget accepting `contentType` and `contentId`
- Create a top-level `showReportBottomSheet(BuildContext context, {required String contentType, required String contentId})` function for easy invocation
- UI layout (following app's bottom sheet patterns like `rating_bottom_sheet.dart`):
  1. Drag handle at top
  2. Title: "Report Content" (or "Report Check-in" / "Report Comment" / "Report Photo" based on contentType)
  3. Reason radio buttons or tappable ListTiles for the 5 reasons: Spam, Harassment, Inappropriate, Copyright, Other
  4. Optional description TextField (max 500 chars, with character counter)
  5. Submit button with loading state
  6. On success: pop the bottom sheet, show SnackBar with backend message ("Report submitted successfully. Our team will review it shortly.")
  7. On error: show SnackBar with error message (handle 409 duplicate gracefully: "You've already reported this content")
- Style: Use `AppTheme.cardDark` for background, `AppTheme.error` for submit button accent, standard border radius 16
- Bottom sheet uses `isScrollControlled: true` and `DraggableScrollableSheet` or `Padding` with `MediaQuery.viewInsets.bottom` for keyboard avoidance
