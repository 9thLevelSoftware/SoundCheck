## Decisions Made

- UserProfileScreen parses response via User.fromJson for type-safe field access to username, avatar, stats
- Placed blocked-users route as child of settings GoRoute (not standalone) for proper shell navigation context
- Co-located userPublicProfileProvider in block_providers.dart since it serves the same screen (UserProfileScreen)
- Block confirmation dialog warns about bilateral blocking and auto-unfollow consequences
