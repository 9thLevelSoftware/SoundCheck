    2. Extend `CacheKeys` in `backend/src/utils/cache.ts`:
       - Add `concertCred: (userId: string) => \`stats:concert-cred:${userId}\``

    3. Create `backend/src/services/StatsService.ts`:
       - Import Database, cache, CacheKeys from existing utils
       - Define `CONCERT_CRED_TTL = 600` (10 minutes per PRFL-08)
       - `getConcertCred(userId: string): Promise<ConcertCred>` -- wraps computeConcertCred with `cache.getOrSet(CacheKeys.concertCred(userId), ..., CONCERT_CRED_TTL)`
       - `private computeConcertCred(userId)` -- runs all 5 queries in parallel via `Promise.all([getBasicStats, getGenreBreakdown, getTopRatedBands, getTopRatedVenues])`, merges into ConcertCred
       - `getBasicStats(userId)` -- single SQL query with scalar subqueries:
         - COUNT(DISTINCT c.id) as totalShows from checkins
         - COUNT(DISTINCT el.band_id) as uniqueBands via checkins JOIN event_lineup
         - COUNT(DISTINCT c.venue_id) as uniqueVenues from checkins
         - COUNT(DISTINCT ub.badge_id) as badgesEarned from user_badges
         - COUNT followers from user_followers WHERE following_id = userId
         - COUNT following from user_followers WHERE follower_id = userId
       - `getGenreBreakdown(userId, limit=5)` -- SQL: checkins JOIN event_lineup JOIN bands, GROUP BY genre, ORDER BY count DESC, LIMIT. Compute percentage in JS from total. Filter WHERE b.genre IS NOT NULL.
       - `getTopRatedBands(userId, limit=5)` -- SQL: checkin_band_ratings JOIN checkins JOIN bands, GROUP BY band, ORDER BY avg_rating DESC then times_seen DESC, LIMIT.
       - `getTopRatedVenues(userId, limit=5)` -- SQL: checkins JOIN venues, WHERE venue_rating > 0, GROUP BY venue, ORDER BY avg_rating DESC then times_visited DESC, LIMIT.
       - `invalidate(userId: string): Promise<void>` -- `cache.del(CacheKeys.concertCred(userId))`

    IMPORTANT: For genre and band queries, always JOIN through event_lineup (checkins -> event_lineup -> bands) rather than through checkins.band_id. This correctly accounts for multi-band events and avoids NULL band_id issues (see Pitfall 5 in RESEARCH.md).

    IMPORTANT: Use `::int` cast on COUNT results and `::numeric(3,2)` on AVG results for proper PostgreSQL type handling.
  </action>
