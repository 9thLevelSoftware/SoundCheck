<objective>
Redesign the Flutter check-in screen from a band-search-first flow to an event-first quick-tap flow. On open, the screen auto-fetches GPS location and calls GET /api/events/nearby to show event cards sorted by distance. User taps an event to check in (single tap = POST /api/checkins with eventId). After check-in, bottom sheets offer optional enrichment: per-set band ratings and venue rating using half-star increments via flutter_rating_bar.

Purpose: This delivers the "check in under 10 seconds" core value proposition -- the entire UX redesign for the primary user action.
Output: Rewritten checkin_screen.dart (event-first), new rating_bottom_sheet.dart, new nearby_event.dart model, updated repository and providers.
</objective>

**checkin_screen.dart -- Full rewrite to event-first flow:**

The screen now has 3 states: (1) Event selection (auto-suggest nearby), (2) Check-in success, (3) optional old band-search fallback.

1. On screen open (`initState`):
   - Auto-fetch GPS position via `LocationService.getCurrentPosition()`
   - Watch `nearbyEventsProvider` which fetches nearby events

2. Main screen structure:
   - AppBar: "Check In" title with close button
   - Body shows event list when GPS succeeds, or loading spinner, or location-unavailable fallback

3. Event list UI (default state):
   - Header: "Shows near you" with distance indicator
   - List of `_EventCard` widgets, each showing:
     - Event name or headliner band name (use `event.eventName ?? event.band?.name ?? 'Unknown Event'`)
     - Venue name and city (`event.venue?.name`)
     - Distance badge ("0.3 km away")
     - Time info (doors/start time if available)
     - Lineup chips (show first 3 bands)
     - Check-in count badge if > 0
   - Each card has a prominent "CHECK IN" button (AppTheme.electricPurple)
   - If no nearby events: show "No shows near you right now" with option to search events manually

4. When user taps "CHECK IN" on event card:
   - Get GPS position (may already be cached from initial fetch)
   - Call `createEventCheckInProvider.submit(eventId: event.id, locationLat: position?.latitude, locationLon: position?.longitude)`
   - Show loading indicator on the tapped button
   - On success: transition to check-in success state

5. Check-in success state:
   - Confetti-style header: "You're checked in!" with event name
   - Show is_verified badge if location was verified
   - Three enrichment option cards:
     - "Rate the bands" -> opens rating_bottom_sheet.dart
     - "Rate the venue" -> opens rating_bottom_sheet.dart (venue tab)
     - "Add photos" -> placeholder (handled in Plan 03)
   - "Done" button returns to previous screen
   - Each enrichment card: icon + label + optional "done" checkmark after completing

6. On duplicate check-in error (409):
   - Show snackbar: "You've already checked in to this event"
   - Navigate to the existing check-in detail screen if possible

7. Location fallback:
   - If GPS unavailable, show search bar for events (query GET /api/events/upcoming with text search)
   - Alternatively show a "Grant location permission" button that calls `LocationService.requestPermission()`

8. Keep existing `_VenueSearchSheet` and `_BandSearchResult` widgets but move them to a "Manual check-in" option accessible via a small text link at bottom: "Can't find your show? Check in manually"
   - Manual check-in uses old flow (bandId + venueId) for backward compat

**Styling notes:**
- Follow existing AppTheme patterns (backgroundDark, surfaceDark, electricPurple, neonPink, liveGreen)
- Event cards: surfaceDark background, rounded corners (16), subtle border
- Distance badge: small chip with AppTheme.liveGreen background
