# Plan 14-01: Onboarding Router + Legal Links + Dark-Only Theme

**Phase**: 14 — Mobile UX & Accessibility
**Requirements**: BETA-10, BETA-11, BETA-12
**Depends on**: Phase 13 complete (no code dependency — parallel-safe)

## Goal

Wire the onboarding flow into the router so first-time users see the carousel before login, fix the App Store–blocking empty legal link handlers, and lock the theme to dark-only for beta.

## Context

- Onboarding carousel, genre picker, `onboardingStateProvider`, and backend sync all exist and work — only the router redirect is missing (BETA-10)
- `pro_feature_screen.dart` has two empty `onPressed: () {}` handlers for Terms/Privacy — App Store rejection risk (BETA-11)
- 46 files hardcode `AppTheme.*Dark` colors — light/system modes render broken UI. Lock to dark-only (BETA-12)

## Tasks

### Task 1: Wire onboarding check into router redirect (BETA-10)

**File**: `mobile/lib/src/core/router/app_router.dart`

**What**: Add `onboardingStateProvider` watch to the redirect function. Insert a check before the auth redirect: if user is not authenticated, has not seen onboarding, and is not already on an onboarding or auth page, redirect to `/onboarding`.

**Redirect flow after change**:
```
Loading → /splash (no change)
!authenticated + !hasSeenOnboarding + !onOnboarding + !onAuth → /onboarding
!authenticated + hasSeenOnboarding + !onAuth → /login
authenticated + onAuth|onOnboarding → /feed
otherwise → allow
```

**Changes**:
1. Import `onboarding_provider.dart` at top of file
2. Inside the `goRouterProvider` body (before `redirect:`), watch `onboardingStateProvider`:
   ```dart
   final onboardingState = ref.watch(onboardingStateProvider);
   final hasSeenOnboarding = onboardingState.valueOrNull ?? true; // default true to avoid blocking on load
   ```
3. In the `redirect` callback, after the `isOnOnboardingPage` check and before the `!isAuthenticated` check, add:
   ```dart
   // First-time user: show onboarding before login
   if (!isAuthenticated && !hasSeenOnboarding && !isOnOnboardingPage && !isOnAuthPage) {
     return '/onboarding';
   }
   ```
4. Add `onboardingStateProvider` to the `refreshListenable` dependencies (the provider is already async, so the router will rebuild when it resolves)

**Lines to modify**: ~58-98 (redirect function)

**No other files need changes** — OnboardingScreen, GenrePickerScreen, and OnboardingProvider already save `hasSeenOnboarding` to SharedPreferences and navigate to `/login` on completion.

### Task 2: Wire Terms of Service and Privacy Policy URLs (BETA-11)

**File**: `mobile/lib/src/features/subscription/presentation/pro_feature_screen.dart`

**What**: Replace the two empty `onPressed: () {}` handlers with `url_launcher` calls.

**Changes**:
1. Add import at top of file:
   ```dart
   import 'package:url_launcher/url_launcher.dart';
   ```
2. Add helper method or use inline:
   ```dart
   Future<void> _launchUrl(String url) async {
     final uri = Uri.parse(url);
     if (await canLaunchUrl(uri)) {
       await launchUrl(uri, mode: LaunchMode.externalApplication);
     }
   }
   ```
3. Line ~183 — Terms of Service handler:
   ```dart
   onPressed: () => _launchUrl('https://soundcheck.app/terms'),
   ```
4. Line ~190 — Privacy Policy handler:
   ```dart
   onPressed: () => _launchUrl('https://soundcheck.app/privacy'),
   ```

**Reference**: `settings_screen.dart` lines 106-116 already uses this exact pattern with `_launchUrl`.

### Task 3: Lock theme to dark-only (BETA-12)

**Three files to modify:**

#### 3a. `mobile/lib/src/core/theme/theme_provider.dart`
- Change default theme from `AppThemeMode.system` to `AppThemeMode.dark` (line ~21)
- In `setTheme()` method, force dark regardless of input (or make setter a no-op)
- Keep `AppThemeMode` enum as-is to avoid breaking imports — just prevent non-dark selection

#### 3b. `mobile/lib/src/features/profile/presentation/settings_screen.dart`
- Replace the 3-option `DropdownButton<AppThemeMode>` (lines ~48-70) with either:
  - A single non-interactive row showing "Dark" as the current theme, or
  - Remove the dropdown entirely and show a `ListTile` with trailing `Text('Dark')` and no interaction
- This prevents users from selecting Light or System

#### 3c. `mobile/lib/main.dart`
- Force `themeMode: ThemeMode.dark` in `MaterialApp.router()` (line ~60-62) instead of the provider-driven value
- Keep `theme:` and `darkTheme:` declarations for compilation — only `themeMode` changes

**No changes to the 46 files with hardcoded dark colors** — they're already correct for dark-only mode.
