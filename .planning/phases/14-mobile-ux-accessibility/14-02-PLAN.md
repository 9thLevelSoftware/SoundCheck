# Plan 14-02: Feed Empty States + Accessibility + Button Contrast

**Phase**: 14 — Mobile UX & Accessibility
**Requirements**: BETA-13, BETA-14, BETA-15
**Depends on**: 14-01 (no hard dependency, but execute in order for clean commits)

## Goal

Give empty feeds actionable CTAs so beta testers aren't stuck on blank screens, add screen reader labels to the bottom nav bar and check-in button, and fix the poor-contrast submit button on owner response sheets.

## Context

- Feed screen uses a custom `_FeedEmptyState` widget instead of the app's `EmptyStateWidget` pattern — no action buttons (BETA-13)
- Bottom nav `_NavItem` and `_CheckInButton` have zero `Semantics` labels — invisible to screen readers (BETA-14)
- Owner response submit button uses `Colors.white` text on `AppTheme.primary` (voltLime) — fails WCAG contrast. Global `ElevatedButtonTheme` correctly uses `backgroundDark` as foreground (BETA-15)

## Tasks

### Task 1: Add actionable empty states to feed tabs (BETA-13)

**File**: `mobile/lib/src/features/feed/presentation/feed_screen.dart`

**What**: Replace the custom `_FeedEmptyState` widget usages with the app's `EmptyStateWidget` pattern, adding action buttons that navigate users to useful screens.

**Changes**:

#### 1a. Friends Tab empty state (lines ~255-260)
Replace with `EmptyStateWidget` using custom params:
```dart
EmptyStateWidget(
  type: EmptyStateType.general,
  customTitle: 'No friend activity yet',
  customMessage: 'Follow friends to see their check-ins here!',
  actionLabel: 'Find Friends',
  onAction: () => context.go('/discover'),
)
```
The "Find Friends" button navigates to the Discover screen where users can search for people.

#### 1b. Happening Now Tab empty state (lines ~359-364)
Replace with `EmptyStateWidget` using context-aware copy:
```dart
EmptyStateWidget(
  type: EmptyStateType.general,
  customTitle: 'No one\'s checked in right now',
  customMessage: 'Check in to a show to be the first! Your friends will see you here when they follow you.',
  actionLabel: 'Explore Events',
  onAction: () => context.go('/discover'),
)
```
Copy explains the feature for users with zero friends instead of implying they should have friends already.

#### 1c. Events Tab empty state (lines ~317-322)
Replace with `EmptyStateWidget`:
```dart
EmptyStateWidget(
  type: EmptyStateType.general,
  customTitle: 'No event activity yet',
  customMessage: 'RSVP to upcoming events to see activity here!',
  actionLabel: 'Discover Events',
  onAction: () => context.go('/discover'),
)
```

#### 1d. Clean up `_FeedEmptyState` widget
After replacing all usages, delete the `_FeedEmptyState` private class (lines ~467-518) since it's no longer referenced.

**Import needed**: `import '../../../../shared/widgets/empty_state_widget.dart';` (if not already imported)

### Task 2: Add Semantics labels to bottom navigation (BETA-14)

**File**: `mobile/lib/src/shared/widgets/scaffold_with_nav_bar.dart`

**What**: Wrap nav items and check-in button with `Semantics` widgets and add `Tooltip` wrappers for accessibility.

**Changes**:

#### 2a. `_NavItem` widget (lines ~137-182)
Wrap the `GestureDetector` in a `Semantics` widget:
```dart
Semantics(
  label: label,  // e.g. "Feed", "Discover", "Profile", "Alerts"
  button: true,
  selected: isSelected,
  child: Tooltip(
    message: label,
    child: GestureDetector(
      // ... existing code
    ),
  ),
)
```

#### 2b. `_CheckInButton` widget (lines ~185-218)
Wrap the `GestureDetector` in `Semantics`:
```dart
Semantics(
  label: 'Check in to a show',
  button: true,
  child: Tooltip(
    message: 'Check In',
    child: GestureDetector(
      // ... existing code
    ),
  ),
)
```

#### 2c. App-wide IconButton tooltips
Do a sweep of `IconButton` usages across the app. Flutter's `IconButton` has a built-in `tooltip` parameter. Add `tooltip:` to any `IconButton` that lacks one. Focus on high-traffic screens:
- Feed screen action bar icons
- App bar leading/trailing icons
- Profile screen action icons
- Check-in screen icons

**Approach**: Search for `IconButton(` across the mobile codebase and add `tooltip:` parameter where missing. The tooltip value should match the icon's semantic purpose (e.g., `tooltip: 'Settings'`, `tooltip: 'Share'`, `tooltip: 'Back'`).

### Task 3: Fix owner response submit button contrast (BETA-15)

**File**: `mobile/lib/src/features/verification/presentation/widgets/owner_response_bottom_sheet.dart`

**What**: Change the submit button text color from `Colors.white` to `AppTheme.backgroundDark` for proper contrast on the voltLime background. Also fix the loading spinner color.

**Changes at lines ~194-203**:

Before:
```dart
color: text.isNotEmpty ? Colors.white : AppTheme.textTertiary,
```

After:
```dart
color: text.isNotEmpty ? AppTheme.backgroundDark : AppTheme.textTertiary,
```

Also fix the `CircularProgressIndicator` color on the same button (loading state):

Before:
```dart
CircularProgressIndicator(color: Colors.white)
```

After:
```dart
CircularProgressIndicator(color: AppTheme.backgroundDark)
```

This matches the global `ElevatedButtonTheme` which correctly uses `foregroundColor: backgroundDark` on `backgroundColor: voltLime` (see `app_theme.dart` lines ~196-214).
