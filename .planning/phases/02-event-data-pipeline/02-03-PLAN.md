<objective>
Enhance user-created events with proper source tracking, add auto-merge logic so Ticketmaster data enriches user-created events, add on-demand Ticketmaster lookup endpoint, and surface event source/status in API responses.

Purpose: Users need to create events for shows not in Ticketmaster (DIY venues, small shows). When Ticketmaster later has the same event, the records should merge rather than duplicate. The on-demand lookup lets the app fetch a specific Ticketmaster event when a user is outside synced coverage. This completes the "events are always available" promise.

Output: Enhanced EventService with user event creation + auto-merge, on-demand lookup endpoint, updated API responses with source/status.
</objective>

         ```sql
         SELECT id FROM events
         WHERE venue_id = $1
           AND event_date::date = $2::date
           AND source = 'user_created'
           AND external_id IS NULL
         LIMIT 1
         ```
         Returns event ID or null.
       - Add method `mergeTicketmasterIntoUserEvent(userEventId: string, tmData: { externalId: string, eventName?: string, ticketUrl?: string, priceMin?: number, priceMax?: number, status?: string })`:
         ```sql
         UPDATE events SET
           external_id = $2,
           source = 'ticketmaster',
           event_name = COALESCE($3, event_name),
           ticket_url = COALESCE($4, ticket_url),
           ticket_price_min = COALESCE($5, ticket_price_min),
           ticket_price_max = COALESCE($6, ticket_price_max),
           status = COALESCE($7, status),
           is_verified = true,
           updated_at = CURRENT_TIMESTAMP
         WHERE id = $1
         ```
         This is the auto-merge: Ticketmaster data enriches user-created event, promotes it to verified.
