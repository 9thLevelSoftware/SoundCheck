```json
"migrate": "node-pg-migrate -j ts --migrations-dir ./migrations --database-url-var DATABASE_URL",
"migrate:up": "npm run migrate up",
"migrate:down": "npm run migrate down",
"migrate:create": "npm run migrate create"
```

Keep existing "migrate" renamed to "migrate:legacy" so the old script is not lost but not the default:


Create the `backend/migrations/` directory if it does not exist.

IMPORTANT: The `-j ts` flag tells node-pg-migrate to use TypeScript migration files. The `--database-url-var DATABASE_URL` flag tells it to read the connection string from the DATABASE_URL environment variable (which is already used by the existing database config).

Write 8 migration files. Each file exports `up` and `down` functions using `MigrationBuilder` from node-pg-migrate. Follow the exact patterns from the RESEARCH.md code examples.

**Migration 001: Setup migration infrastructure**
- Enable `pg_trgm` extension (needed for Phase 2 band name matching, install now to avoid future DDL).
- Detect and handle the old `events` table from `migrate-events-model.ts`. If an `events` table exists AND has a `band_id` column (wrong schema from the old migration), drop it with CASCADE. This also drops `checkin_toasts` and `checkin_comments` if they were created by that old migration (the correct `checkin_comments` from `database-schema.sql` will be unaffected since it references `checkins` which still exists).
- Use conditional DDL:
  ```sql
  DO $$ BEGIN
    IF EXISTS (
      SELECT 1 FROM information_schema.columns
      WHERE table_name = 'events' AND column_name = 'band_id'
    ) THEN
      DROP TABLE IF EXISTS checkin_toasts CASCADE;
      DROP TABLE IF EXISTS events CASCADE;
    END IF;
  END $$;
  ```
- The `down` function should be a no-op (cannot un-detect old schema).

**Migration 002: Create events table**
- Create `events` table per RESEARCH.md example. Columns:
  - `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
  - `venue_id UUID NOT NULL REFERENCES venues(id) ON DELETE CASCADE`
  - `event_date DATE NOT NULL`
  - `event_name VARCHAR(255)`
  - `description TEXT`
  - `doors_time TIME`
  - `start_time TIME`
  - `end_time TIME`
  - `ticket_url VARCHAR(500)`
  - `ticket_price_min DECIMAL(10,2)`
  - `ticket_price_max DECIMAL(10,2)`
  - `is_sold_out BOOLEAN DEFAULT FALSE`
  - `is_cancelled BOOLEAN DEFAULT FALSE`
  - `event_type VARCHAR(50) DEFAULT 'concert'`
  - `source VARCHAR(50) DEFAULT 'user_created'`
  - `external_id VARCHAR(255)`
  - `created_by_user_id UUID REFERENCES users(id)`
  - `is_verified BOOLEAN DEFAULT FALSE`
  - `total_checkins INTEGER DEFAULT 0`
  - `created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP NOT NULL`
  - `updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP NOT NULL`
- Add unique constraint: `UNIQUE(source, external_id)` named `unique_external_event`
- Add indexes: `venue_id`, `event_date`, `(venue_id, event_date)`, `source`
- Add `updated_at` trigger using existing `update_updated_at_column()` function
- `down`: drop table with CASCADE

**Migration 003: Create event_lineup junction table**
- Create `event_lineup` table per RESEARCH.md example. Columns:
  - `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
  - `event_id UUID NOT NULL REFERENCES events(id) ON DELETE CASCADE`
  - `band_id UUID NOT NULL REFERENCES bands(id) ON DELETE CASCADE`
  - `set_order INTEGER NOT NULL DEFAULT 0`
  - `set_time TIME`
  - `is_headliner BOOLEAN DEFAULT FALSE`
  - `created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP NOT NULL`
- Add unique constraint: `UNIQUE(event_id, band_id)` named `unique_lineup_slot`
- Add indexes: `event_id`, `band_id`, `(event_id, set_order)`
- `down`: drop table with CASCADE

**Migration 004: Add event_id and new columns to checkins**
- Add columns to existing `checkins` table:
  - `event_id UUID REFERENCES events(id) ON DELETE CASCADE` (NULLABLE -- existing rows have no event)
  - `venue_rating DECIMAL(2,1)` with CHECK: `venue_rating IS NULL OR (venue_rating >= 0.5 AND venue_rating <= 5.0)`
  - `review_text TEXT`
  - `image_urls TEXT[]`
  - `is_verified BOOLEAN DEFAULT FALSE`
- Add partial unique index: `UNIQUE(user_id, event_id) WHERE event_id IS NOT NULL` named `idx_unique_user_event_checkin`
- Add index on `event_id`
- `down`: drop the index and columns

**Migration 005: Create checkin_band_ratings table**
- Per RESEARCH.md example. Columns:
  - `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
  - `checkin_id UUID NOT NULL REFERENCES checkins(id) ON DELETE CASCADE`
  - `band_id UUID NOT NULL REFERENCES bands(id) ON DELETE CASCADE`
  - `rating DECIMAL(2,1) NOT NULL` with CHECK: `rating >= 0.5 AND rating <= 5.0`
  - `created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP NOT NULL`
- Add unique constraint: `UNIQUE(checkin_id, band_id)` named `unique_checkin_band_rating`
- Add indexes: `checkin_id`, `band_id`
- `down`: drop table with CASCADE

**Migration 006: Add timezone to venues**
- Add `timezone VARCHAR(50)` column to `venues` table with comment 'IANA timezone identifier, e.g. America/New_York'
- `down`: drop column

**Migration 007: Add event_id to notifications**
- Add `event_id UUID REFERENCES events(id) ON DELETE CASCADE` to `notifications` table (NULLABLE)
- Add index on `event_id`
- `down`: drop column and index

**Migration 008: Add JSONB badge criteria**
- Add `criteria JSONB DEFAULT '{}'::jsonb` column to `badges` table with comment
- Add GIN index on `criteria`
- `down`: drop column

IMPORTANT CONSTRAINTS:
- All rating columns MUST be DECIMAL(2,1), NOT INTEGER. Half-star increments require decimal.
- All timestamps MUST be TIMESTAMPTZ (already the case for existing columns, enforce for new ones).
- event_id on checkins MUST be nullable (existing rows have no event).
- Do NOT modify or drop any existing columns. This is expand-only.
- Do NOT touch the `reviews` table. BadgeService depends on it until Phase 4.
- Each migration file must import `type { MigrationBuilder }` from 'node-pg-migrate' (use type-only import).

After writing all 8 files, run the migrations:
```bash
cd backend && npm run migrate:up
```

If DATABASE_URL is not set in the environment, the command will fail. In that case, set up a local .env file with DATABASE_URL or use individual DB_* vars, then retry. If no database is available, at minimum verify the migration files are syntactically valid by running:
```bash
cd backend && npx tsc --noEmit migrations/*.ts
```
