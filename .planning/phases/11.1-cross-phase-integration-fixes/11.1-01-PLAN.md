<objective>
Create PostgreSQL triggers to automatically maintain denormalized toast_count and comment_count on the checkins table, and backfill existing data.

Purpose: Phase 11 Plan 02 added toast_count and comment_count columns to checkins (migration 024) and switched feed queries to read these columns instead of COUNT(DISTINCT) joins. But no write path was ever implemented -- the columns are permanently 0. This migration adds database-level triggers so every INSERT/DELETE on toasts or checkin_comments automatically updates the parent checkin's count. A backfill UPDATE corrects existing data.

Output: Migration 037 with trigger functions, triggers, and backfill query.
</objective>

Create migration file `backend/migrations/037_denormalized-count-triggers.ts` using the node-pg-migrate pattern.

In the `up` function, execute these SQL statements via `pgm.sql()`:

1. **Toast count trigger function:** Create `update_checkin_toast_count()` RETURNS TRIGGER function. On INSERT: `UPDATE checkins SET toast_count = toast_count + 1 WHERE id = NEW.checkin_id`. On DELETE: `UPDATE checkins SET toast_count = GREATEST(toast_count - 1, 0) WHERE id = OLD.checkin_id`. Use LANGUAGE plpgsql. Use `CREATE OR REPLACE FUNCTION`.

2. **Toast triggers:** Create two triggers on the `toasts` table:
   - `trg_toast_count_insert`: AFTER INSERT, FOR EACH ROW, EXECUTE FUNCTION update_checkin_toast_count()
   - `trg_toast_count_delete`: AFTER DELETE, FOR EACH ROW, EXECUTE FUNCTION update_checkin_toast_count()

3. **Comment count trigger function:** Create `update_checkin_comment_count()` RETURNS TRIGGER function. On INSERT: `UPDATE checkins SET comment_count = comment_count + 1 WHERE id = NEW.checkin_id`. On DELETE: `UPDATE checkins SET comment_count = GREATEST(comment_count - 1, 0) WHERE id = OLD.checkin_id`. Use LANGUAGE plpgsql. Use `CREATE OR REPLACE FUNCTION`.

4. **Comment triggers:** Create two triggers on the `checkin_comments` table:
   - `trg_comment_count_insert`: AFTER INSERT, FOR EACH ROW, EXECUTE FUNCTION update_checkin_comment_count()
   - `trg_comment_count_delete`: AFTER DELETE, FOR EACH ROW, EXECUTE FUNCTION update_checkin_comment_count()

5. **Backfill existing data:** Run a single UPDATE to set correct counts for all existing checkins:
   ```sql
   UPDATE checkins SET
     toast_count = (SELECT COUNT(*) FROM toasts WHERE checkin_id = checkins.id),
     comment_count = (SELECT COUNT(*) FROM checkin_comments WHERE checkin_id = checkins.id);
   ```

In the `down` function:
- DROP TRIGGER IF EXISTS for all 4 triggers (on their respective tables)
- DROP FUNCTION IF EXISTS for both trigger functions
- Reset counts: `UPDATE checkins SET toast_count = 0, comment_count = 0`

IMPORTANT: Use `GREATEST(count - 1, 0)` for DELETE operations to floor at 0 and prevent negative counts from data inconsistencies.
