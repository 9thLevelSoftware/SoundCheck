**PostgreSQL triggers on toasts and checkin_comments tables to automatically maintain checkins.toast_count and checkins.comment_count, with backfill of existing data**

## Accomplishments
- Created trigger functions `update_checkin_toast_count()` and `update_checkin_comment_count()` using plpgsql
- Added 4 AFTER triggers (INSERT+DELETE on toasts, INSERT+DELETE on checkin_comments) for automatic count maintenance
- Backfill query corrects all existing checkins with accurate counts from correlated subqueries
- Down migration cleanly reverses all triggers, functions, and resets counts to 0

## Files Created/Modified
- `backend/migrations/037_denormalized-count-triggers.ts` - Migration with 2 trigger functions, 4 triggers, backfill UPDATE, and complete down migration

## Decisions Made
- Used `GREATEST(count - 1, 0)` in DELETE triggers to floor at 0 and prevent negative counts from potential data inconsistencies
- Separate INSERT and DELETE triggers per table (not combined INSERT OR DELETE trigger) for clarity
- Backfill uses correlated subqueries (`SELECT COUNT(*) FROM toasts WHERE checkin_id = checkins.id`) for accurate one-pass update
