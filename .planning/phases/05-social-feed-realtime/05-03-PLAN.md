    Create feed_providers.dart with Riverpod providers for feed state management. Use @riverpod annotation (code-gen).

    **1. Friends feed provider with cursor pagination:**
    ```dart
    @riverpod
    class FriendsFeedNotifier extends _$FriendsFeedNotifier {
      String? _nextCursor;
      bool _hasMore = true;
      List<FeedItem> _items = [];

      @override
      Future<List<FeedItem>> build() async {
        _nextCursor = null;
        _hasMore = true;
        _items = [];
        return _fetchPage();
      }

      Future<List<FeedItem>> _fetchPage() async {
        final repo = ref.read(feedRepositoryProvider);
        final page = await repo.getFriendsFeed(cursor: _nextCursor);
        _nextCursor = page.nextCursor;
        _hasMore = page.hasMore;
        _items = [..._items, ...page.items];
        return _items;
      }

      Future<void> loadMore() async {
        if (!_hasMore) return;
        state = AsyncValue.data(await _fetchPage());
      }

      void prependItems(List<FeedItem> newItems) {
        _items = [...newItems, ..._items];
        state = AsyncValue.data(_items);
      }
    }
    ```

    **2. Event feed provider (parameterized by eventId):**
    Similar cursor pagination pattern but takes eventId as family parameter.

    **3. Happening Now provider:**

    @riverpod

      void reset() => state = 0;

    - On each new_checkin event, invalidates unseenCountsProvider

    This listener should be initialized when the feed screen is active. Use a ConsumerStatefulWidget initState or a provider that auto-disposes.
