**searchBands(query: string, limit: number)** — private method:
```sql
WITH fts_results AS (
  SELECT id, name, genre, genres, hometown, description, image_url, is_active,
    ts_rank(search_vector, websearch_to_tsquery('english', $1)) AS rank
  FROM bands
  WHERE search_vector @@ websearch_to_tsquery('english', $1)
    AND is_active = TRUE
  ORDER BY rank DESC
  LIMIT $2
),
fuzzy_results AS (
  SELECT id, name, genre, genres, hometown, description, image_url, is_active,
    similarity(name, $1) AS rank
  FROM bands
  WHERE similarity(name, $1) > 0.3
    AND is_active = TRUE
    AND id NOT IN (SELECT id FROM fts_results)
  ORDER BY rank DESC
  LIMIT $2
)
SELECT * FROM fts_results
UNION ALL
SELECT * FROM fuzzy_results
LIMIT $2;
```
Map rows to Band interface using existing dbMappers pattern or inline mapping.

**searchVenues(query, limit)** — same pattern using venues.search_vector, fuzzy on venues.name.

**searchEvents(query, limit)** — same pattern but:
- JOIN venues v ON e.venue_id = v.id to include venue info

- Fuzzy fallback matches on event_name OR joined venue name
