**Migration 033 — Verification claims + claimed profiles:**
1. Create `verification_claims` table:
   - `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
   - `user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE`
   - `entity_type VARCHAR(10) NOT NULL CHECK (entity_type IN ('venue', 'band'))`
   - `entity_id UUID NOT NULL`
   - `status VARCHAR(10) NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'approved', 'denied'))`
   - `evidence_text TEXT` (claimant's justification)
   - `evidence_url VARCHAR(500)` (link to website/social proof)
   - `reviewed_by UUID REFERENCES users(id) ON DELETE SET NULL`
   - `reviewed_at TIMESTAMPTZ`
   - `review_notes TEXT`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP`
2. Create partial unique index: `CREATE UNIQUE INDEX idx_claims_one_pending ON verification_claims (entity_type, entity_id) WHERE status = 'pending'` — enforces only one pending claim per entity
3. Create indexes: `idx_claims_user_id`, `idx_claims_entity`, `idx_claims_status`
4. Add `claimed_by_user_id UUID REFERENCES users(id) ON DELETE SET NULL` to both `venues` and `bands` tables
5. Create `wilson_lower_bound(positive BIGINT, total BIGINT) RETURNS DOUBLE PRECISION` SQL function (IMMUTABLE, plpgsql) — uses z=1.96, guards n=0 with RETURN 0. See RESEARCH.md for exact formula.
6. Add updated_at trigger for verification_claims

**Migration 034 — tsvector search columns + GIN indexes:**
1. Add `search_vector tsvector GENERATED ALWAYS AS (setweight(to_tsvector('english', COALESCE(name, '')), 'A') || setweight(to_tsvector('english', COALESCE(genre, '')), 'B') || setweight(to_tsvector('english', COALESCE(hometown, '')), 'C') || setweight(to_tsvector('english', COALESCE(description, '')), 'D')) STORED` to `bands` table
2. Add `search_vector tsvector GENERATED ALWAYS AS (setweight(to_tsvector('english', COALESCE(name, '')), 'A') || setweight(to_tsvector('english', COALESCE(city, '')), 'B') || setweight(to_tsvector('english', COALESCE(state, '')), 'C') || setweight(to_tsvector('english', COALESCE(description, '')), 'D')) STORED` to `venues` table
3. Add `search_vector tsvector GENERATED ALWAYS AS (setweight(to_tsvector('english', COALESCE(event_name, '')), 'A')) STORED` to `events` table
4. Create GIN indexes: `idx_bands_search_vector`, `idx_venues_search_vector`, `idx_events_search_vector`
5. IMPORTANT: Use `GENERATED ALWAYS AS ... STORED` (not VIRTUAL). GIN indexes require materialized data.
6. Down migration: DROP the columns (indexes drop automatically)

**Migration 035 — Genre array migration:**
1. Add `genres TEXT[]` column to `bands` table (nullable initially)
2. Backfill: `UPDATE bands SET genres = CASE WHEN genre IS NOT NULL AND genre != '' THEN string_to_array(genre, ', ') ELSE ARRAY[]::TEXT[] END` — handles both single genres and potential comma-separated values
3. Set default: `ALTER TABLE bands ALTER COLUMN genres SET DEFAULT ARRAY[]::TEXT[]`
4. Create GIN index: `CREATE INDEX idx_bands_genres ON bands USING GIN (genres)` for array containment queries
5. Do NOT drop or alter the old `genre` column — keep for backward compatibility during transition
6. Down migration: DROP genres column and index

**Migration 036 — Review owner response:**
1. Add `owner_response TEXT` to `reviews` table
2. Add `owner_response_at TIMESTAMPTZ` to `reviews` table
3. Down migration: DROP both columns
