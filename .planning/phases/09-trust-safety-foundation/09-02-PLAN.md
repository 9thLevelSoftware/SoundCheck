**ReportService.ts:**
- Class with constructor taking optional `db` parameter (default `Database.getInstance()`), same pattern as existing services.
- `createReport(reporterId: string, data: CreateReportRequest): Promise<Report>` — Validates content exists (query checkins/checkin_comments/checkin_photos tables based on contentType). Inserts into `reports` table. The UNIQUE constraint handles dedup (catch unique violation error and return a clear "already reported" message). Resolves `target_user_id` from the content's author. If contentType is 'photo', enqueue a SafeSearch scan job (import moderationQueue, check for null before adding). Returns the created report.
- `getReportsForContent(contentType: ContentType, contentId: string): Promise<Report[]>` — Returns all reports for a piece of content.
- `getUserReportCount(userId: string, since: Date): Promise<number>` — Used for rate limiting (10/day). SELECT COUNT(*) FROM reports WHERE reporter_id = $1 AND created_at > $2.

**ModerationService.ts:**
- Class with constructor taking optional `db` parameter.
- `createModerationItem(data: { contentType, contentId, source, reportId?, safesearchResults? }): Promise<ModerationItem>` — Inserts into moderation_items table.
- `autoHideContent(contentType: ContentType, contentId: string): Promise<void>` — Soft-hide content. For checkins: `UPDATE checkins SET is_hidden = true WHERE id = $1`. For comments: `UPDATE checkin_comments SET is_hidden = true WHERE id = $1`. NOTE: If `is_hidden` columns don't exist yet, add them via ALTER TABLE in migration 026 (append to the up function).
- `getPendingItems(page: number, limit: number): Promise<{ items: ModerationItem[], total: number }>` — Query moderation_items WHERE status = 'pending_review' ORDER BY created_at ASC with pagination.
- `reviewItem(itemId: string, adminId: string, action: 'approved' | 'removed' | 'user_warned'): Promise<ModerationItem>` — Updates moderation_items with reviewed_by, reviewed_at, action_taken, status = 'reviewed'. If action is 'approved', unhide the content (set is_hidden = false). If 'removed', keep hidden and also update related reports to status = 'actioned'.
- `getItemById(itemId: string): Promise<ModerationItem | null>` — Fetch single moderation item with details.

**ImageModerationService.ts:**
- Follow the exact code example from the research doc (see 09-RESEARCH.md "SafeSearch Image Scanning Service").
- Class with constructor that creates `vision.ImageAnnotatorClient()`. Use graceful degradation: if `@google-cloud/vision` import fails or credentials missing, log a warning and set a `configured` flag to false.
- `scanImage(imageUrl: string): Promise<SafeSearchResult>` — Returns `{ isFlagged, annotations, flagReasons }`. If not configured, return `{ isFlagged: false, annotations: {}, flagReasons: [] }`.
- Flag thresholds: LIKELY or VERY_LIKELY for adult/violence, VERY_LIKELY only for racy (concerts have suggestive but not explicit content).

**moderationQueue.ts:**
- Follow the exact same pattern as `badgeQueue.ts`: import Queue from bullmq, createBullMQConnection from config/redis. Create `moderationQueue` with graceful degradation (null if Redis not configured). Queue name: 'image-moderation'. Default job options: attempts 3, backoff exponential 1000ms, removeOnComplete count 100, removeOnFail count 200.

**moderationWorker.ts:**
- Follow the same pattern as `badgeWorker.ts`: import Worker from bullmq. Create worker for 'image-moderation' queue.
- Job handler: Extract `{ contentType, contentId, imageUrl, userId }` from job.data. Instantiate ImageModerationService and ModerationService. Call `imageMod.scanImage(imageUrl)`. If `result.isFlagged`, call `moderationService.autoHideContent(contentType, contentId)` then `moderationService.createModerationItem({ contentType, contentId, source: 'auto_safesearch', safesearchResults: result.annotations })`. Log results via logInfo/logWarn.
- Graceful degradation: if Redis not configured, don't create worker, log warning.

**ReportController.ts:**
- Class following existing controller pattern (arrow function methods, ApiResponse format).
- `createReport = async (req, res)` — Extract user from req.user. Check rate limit: call reportService.getUserReportCount(userId, 24h ago), reject with 429 if >= 10. Validate body against CreateReportRequest. Call reportService.createReport(). Return 201 with success: true.
- `getModerationQueue = async (req, res)` — For admins. Parse page/limit from query. Call moderationService.getPendingItems(). Return 200 with items and total.
- `reviewModerationItem = async (req, res)` — For admins. Extract itemId from params, action from body. Call moderationService.reviewItem(). Return 200 with updated item.
