**EmailService.ts:**
- Follow the code example from research doc (see 09-RESEARCH.md "Password Reset Email with Resend").
- Class with constructor that reads RESEND_API_KEY from env. If not set, set `configured = false` and log a warning (graceful degradation — app still starts, password reset disabled).
- `sendPasswordResetEmail(to: string, resetToken: string): Promise<void>` — If not configured, log warning and return (don't throw). Construct reset URL using `soundcheck://reset-password?token=${resetToken}` scheme. Send via Resend with branded HTML template (SoundCheck green #CCFF00 accent, simple layout with "Reset Password" button). Subject: "Reset your SoundCheck password".
- `isConfigured(): boolean` — Returns whether Resend is configured.

**PasswordResetService.ts:**
- Class with constructor taking optional `db` parameter and optional `emailService` parameter.
- `requestReset(email: string): Promise<{ sent: boolean, message: string }>` — Normalize email to lowercase. Look up user by email. If not found, return `{ sent: false, message: 'If an account exists...' }` (don't reveal whether email exists). Check if user is social-auth-only: query `SELECT password FROM users WHERE email = $1`, if password starts with `$SOCIAL_AUTH$`, return `{ sent: false, message: 'This account uses Google/Apple Sign-In. Please use that method to log in.' }`. Revoke all existing reset tokens for this user: `UPDATE password_reset_tokens SET used_at = CURRENT_TIMESTAMP WHERE user_id = $1 AND used_at IS NULL`. Generate new token using same pattern as refresh tokens: `crypto.randomBytes(32).toString('hex')`, store SHA-256 hash in `password_reset_tokens` with 1-hour expiry. Call emailService.sendPasswordResetEmail(email, rawToken). Return `{ sent: true, message: 'If an account exists...' }`.
- `resetPassword(token: string, newPassword: string): Promise<void>` — Hash the submitted token with SHA-256. Look up in password_reset_tokens WHERE token_hash = hash AND used_at IS NULL AND expires_at > NOW(). If not found, throw "Invalid or expired reset token". Hash new password with bcryptjs (same as registration flow). Update users SET password = newHash WHERE id = userId. Mark token as used: UPDATE password_reset_tokens SET used_at = CURRENT_TIMESTAMP WHERE id = tokenId. Revoke all refresh tokens for this user (forces re-login on all devices): DELETE FROM refresh_tokens WHERE user_id = userId (or use the existing revokeAllUserTokens pattern from auth.ts).
- Validate new password meets existing requirements (min 8 chars, at least one uppercase — same rules as registration in validationSchemas.ts).

**PasswordResetController.ts:**
- Class following existing controller pattern.
- `forgotPassword = async (req, res)` — Extract email from body. Call passwordResetService.requestReset(email). Return 200 with message (always same message whether email exists or not — prevent enumeration).
- `resetPassword = async (req, res)` — Extract token and newPassword from body. Call passwordResetService.resetPassword(token, newPassword). Return 200 with { success: true, message: 'Password reset successful. Please log in.' }.

**passwordResetRoutes.ts:**
- Router with:
  - POST '/forgot-password' — rate limited (5 per hour per IP using existing perUserRateLimit with keyPrefix 'pwd-reset'), passwordResetController.forgotPassword
  - POST '/reset-password' — rate limited (5 per hour per IP), passwordResetController.resetPassword
- Zod validation: forgotPassword body requires `email` (valid email string). resetPassword body requires `token` (string, min 64 chars hex) and `newPassword` (string, min 8 chars, at least one uppercase).

**index.ts:**
- Import passwordResetRoutes. Mount at `app.use('/api/auth', passwordResetRoutes);`
