SQL: `SELECT AVG(cbr.rating)::numeric(3,2), COUNT(DISTINCT cbr.id)::int, COUNT(DISTINCT c.user_id)::int FROM checkin_band_ratings cbr JOIN checkins c ON cbr.checkin_id = c.id WHERE cbr.band_id = $1`. Returns zeros if no ratings.
       - `getVenueAggregateRating(venueId: string): Promise<VenueAggregate>` -- Same pattern. SQL: `SELECT AVG(c.venue_rating)::numeric(3,2), COUNT(DISTINCT c.id)::int, COUNT(DISTINCT c.user_id)::int FROM checkins c WHERE c.venue_id = $1 AND c.venue_rating IS NOT NULL AND c.venue_rating > 0`.
       - Import Database, cache, CacheKeys from existing modules following StatsService pattern.

    CRITICAL: Do NOT compute aggregates from bands.average_rating or venues.average_rating (those use the old reviews table). Always use checkin_band_ratings.rating for bands and checkins.venue_rating for venues.

    5. **Enhance BandController.getBandById**: After getting band data, also call `discoveryService.getBandAggregateRating(id)` and `eventService.getEventsByBand(id, { upcoming: true, limit: 5 })` in parallel (Promise.all). Merge aggregate + upcoming into response: `{ ...band, aggregate: bandAggregate, upcomingShows: events }`. Create a private `discoveryService = new DiscoveryService()` and `eventService = new EventService()` on the controller.

    6. **Enhance VenueController.getVenueById**: Same pattern -- call `discoveryService.getVenueAggregateRating(id)` and `eventService.getEventsByVenue(id, { upcoming: true, limit: 5 })` in parallel. Merge into response: `{ ...venue, aggregate: venueAggregate, upcomingEvents: events }`.

    7. **Add cache invalidation to CheckinService**: In the existing fire-and-forget invalidation blocks (after rating submission in the PATCH /ratings handler, and after check-in creation):
       - After band ratings are set: `cache.del(CacheKeys.bandAggregate(bandId)).catch(...)` for each band rated
       - After venue rating is set: `cache.del(CacheKeys.venueAggregate(venueId)).catch(...)`
       - After check-in delete: same invalidation pattern

    Keep all invalidation fire-and-forget (non-blocking, errors logged but never thrown). Follow the existing pattern at lines ~260-268 of CheckinService.ts.
