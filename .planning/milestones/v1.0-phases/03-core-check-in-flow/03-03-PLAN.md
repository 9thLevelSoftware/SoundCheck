**R2Service.ts -- New service:**

```typescript
import { S3Client, PutObjectCommand, DeleteObjectCommand } from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';
import crypto from 'crypto';
```

1. Constructor:
   - `this.bucket = process.env.R2_BUCKET_NAME || 'soundcheck-photos'`
   - `this.publicUrl = process.env.R2_PUBLIC_URL || ''`
   - `this.isConfigured` flag: true only if CLOUDFLARE_ACCOUNT_ID AND R2_ACCESS_KEY_ID AND R2_SECRET_ACCESS_KEY are all present
   - If configured, create S3Client:
     ```
     new S3Client({
       region: 'auto',
       endpoint: `https://${process.env.CLOUDFLARE_ACCOUNT_ID}.r2.cloudflarestorage.com`,
       credentials: {
         accessKeyId: process.env.R2_ACCESS_KEY_ID || '',
         secretAccessKey: process.env.R2_SECRET_ACCESS_KEY || '',
       },
     })
     ```
   - If not configured, log warning: 'R2Service: Missing R2 credentials, photo uploads disabled'

2. `async getPresignedUploadUrl(contentType: string, prefix: string = 'checkins')`:
   - If not configured, throw Error('Photo uploads not configured')
   - Validate contentType is image/* (image/jpeg, image/png, image/webp, image/heic)
   - Generate objectKey: `${prefix}/${crypto.randomBytes(16).toString('hex')}.${ext}` where ext from contentType split
   - Generate presigned PUT URL with 600s expiry (10 minutes):
     ```
     const uploadUrl = await getSignedUrl(this.s3, new PutObjectCommand({
       Bucket: this.bucket,
       Key: objectKey,
       ContentType: contentType,
     }), { expiresIn: 600 });
     ```
   - Return `{ uploadUrl, objectKey, publicUrl: \`${this.publicUrl}/${objectKey}\` }`

3. `async deleteObject(objectKey: string)`:
   - If not configured, return silently
   - `await this.s3.send(new DeleteObjectCommand({ Bucket: this.bucket, Key: objectKey }))`

**CheckinService.ts -- Add photo methods:**

1. `async addPhotos(checkinId: string, userId: string, photoKeys: string[])`:
   - Verify checkin belongs to userId
   - Validate photoKeys length: max 4 total (check existing image_urls + new)
   - Get existing image_urls from checkin: `SELECT image_urls FROM checkins WHERE id = $1 AND user_id = $2`
   - Combine existing + new, ensure total <= 4
   - Build public URLs from keys: `photoKeys.map(key => \`${process.env.R2_PUBLIC_URL}/${key}\`)`
   - `UPDATE checkins SET image_urls = $1 WHERE id = $2` (replace entire array with combined)
   - Return updated checkin via getCheckinById

2. `async requestPhotoUploadUrls(checkinId: string, userId: string, contentTypes: string[])`:
   - Verify checkin belongs to userId
   - Check existing photo count + requested count <= 4
   - For each contentType, call `r2Service.getPresignedUploadUrl(contentType, \`checkins/${checkinId}\`)`
   - Return array of `{ uploadUrl, objectKey, publicUrl }`

**upload_repository.dart -- New file:**

1. `UploadRepository` class with DioClient dependency (same pattern as CheckInRepository)

2. `Future<List<PresignedUpload>> requestPresignedUrls(String checkinId, List<String> contentTypes)`:
   - POST `${ApiConfig.checkins}/$checkinId/photos` with body `{ contentTypes }`
   - Parse response into `List<PresignedUpload>` where PresignedUpload = `{ uploadUrl, objectKey, publicUrl }`

3. `Future<void> uploadPhotoToR2(String presignedUrl, Uint8List photoBytes, String contentType)`:
   - Use `Dio()` (fresh instance, NOT the authenticated DioClient -- presigned URL is self-authenticating)
   - PUT to presignedUrl with:
     ```dart
     await Dio().put(
       presignedUrl,
       data: Stream.fromIterable([photoBytes]),
       options: Options(
         headers: {
           'Content-Type': contentType,
           'Content-Length': photoBytes.length,
         },
       ),
     );
     ```
   - CRITICAL: Content-Type header MUST match what was used to generate the presigned URL

4. `Future<CheckIn> confirmPhotoUploads(String checkinId, List<String> photoKeys)`:
   - PATCH `${ApiConfig.checkins}/$checkinId/photos` with body `{ photoKeys }`
   - Parse response as CheckIn

5. `Future<CheckIn?> uploadPhotos(String checkinId, List<XFile> photos)` -- convenience method:
   - Determine content types from photos (XFile.mimeType or default to 'image/jpeg')
   - Request presigned URLs for all photos
   - For each photo:
     - Compress using flutter_image_compress: `FlutterImageCompress.compressWithFile(photo.path, quality: 85, minWidth: 1920, minHeight: 1080)`
     - Upload compressed bytes to R2 via presigned URL
   - Confirm all uploads with PATCH
   - Return updated CheckIn
