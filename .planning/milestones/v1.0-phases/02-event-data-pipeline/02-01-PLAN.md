  <action>
    1. Create `backend/src/services/TicketmasterAdapter.ts`:
       - Class with private axios instance configured with baseURL `https://app.ticketmaster.com/discovery/v2` and apikey param from `TICKETMASTER_API_KEY` env var.
       - Constructor throws if `TICKETMASTER_API_KEY` is not set.
       - Set axios timeout to 10000ms.
       - Track daily API call count via a simple in-memory counter (reset at midnight UTC). Log warning when approaching 4500 calls. Refuse calls at 4900 (reserve 100 for on-demand lookups).
       - **searchMusicEvents(params)**: Accepts { latlong: string, radius: number, startDateTime: string, endDateTime: string, size?: number (default 200), page?: number (default 0) }. Calls `GET /events.json` with classificationName=music, sort=date,asc, unit=miles. Returns { events: TicketmasterEvent[], page: { totalElements, totalPages, number, size } }. Handle empty responses (_embedded?.events may be missing when 0 results).
       - **fetchAllEventsForRegion(latlong, radius, startDate, endDate)**: Paginates through all pages (max 5 pages = 1000 items). If totalElements > 1000, subdivide the date range into halves and recurse (date-range subdivision). Returns NormalizedEvent[]. Calls `normalizeEvent()` on each raw TM event.
       - **getEventById(eventId)**: Single event fetch. Returns TicketmasterEvent | null. 404 returns null.
       - **normalizeEvent(tmEvent)**: Converts raw TicketmasterEvent to NormalizedEvent. Skip events without `_embedded?.venues?.[0]` (log and skip). Extract venue from first _embedded.venues entry. Extract attractions from _embedded.attractions (may be empty array). Map status code (onsale/offsale -> active, canceled -> cancelled, postponed -> postponed, rescheduled -> rescheduled).
       - Add 200ms delay between API calls (stay safely under 5/sec limit). Use simple `await new Promise(r => setTimeout(r, 200))` between pagination calls.
       - Follow the existing adapter pattern from MusicBrainzService (axios instance, error handling, response parsing).

    2. Create `backend/src/services/BandMatcher.ts`:
       - Uses Database.getInstance() for queries (same pattern as BandService, VenueService).
       - **matchOrCreateBand(name, externalId?, genre?, imageUrl?)** returns Promise<{ bandId: string, matchType: 'external_id' | 'exact' | 'fuzzy' | 'created', score?: number }>
       - Matching cascade:
         Step 1: If externalId provided, query `SELECT id FROM bands WHERE source = 'ticketmaster' AND external_id = $1`. If found, return { bandId, matchType: 'external_id' }.
         Step 2: Exact case-insensitive match: `SELECT id FROM bands WHERE LOWER(name) = LOWER($1) AND is_active = true LIMIT 1`. If found AND externalId provided, update the band's external_id+source (only if currently NULL). Return { bandId, matchType: 'exact' }.
         Step 3: Fuzzy match with pg_trgm: `SELECT id, name, similarity(LOWER(name), LOWER($1)) AS score FROM bands WHERE LOWER(name) % LOWER($1) AND similarity(LOWER(name), LOWER($1)) >= 0.8 AND is_active = true ORDER BY score DESC LIMIT 1`. If found, update external_id if provided. Return { bandId, matchType: 'fuzzy', score }.
         Step 4: No match -- create new band: `INSERT INTO bands (name, genre, image_url, source, external_id) VALUES ($1, $2, $3, 'ticketmaster', $4) RETURNING id`. Return { bandId, matchType: 'created' }.
       - **matchOrCreateVenue(tmVenue: NormalizedEvent['venue'])** returns Promise<{ venueId: string, isNew: boolean }>
         Step 1: If externalId provided, upsert: `INSERT INTO venues (name, address, city, state, country, postal_code, latitude, longitude, timezone, source, external_id) VALUES (...) ON CONFLICT (source, external_id) WHERE external_id IS NOT NULL DO UPDATE SET name = EXCLUDED.name, address = COALESCE(EXCLUDED.address, venues.address), latitude = COALESCE(EXCLUDED.latitude, venues.latitude), longitude = COALESCE(EXCLUDED.longitude, venues.longitude), updated_at = CURRENT_TIMESTAMP RETURNING id, (xmax = 0) AS is_new`.
         Step 2: If no externalId, fall back to name+city match: `SELECT id FROM venues WHERE LOWER(name) = LOWER($1) AND LOWER(city) = LOWER($2) LIMIT 1`.
         Step 3: If no match, insert new venue.
       - Note: venue matching is in BandMatcher for simplicity (it handles all entity resolution). The class could be named EntityMatcher but BandMatcher is clearer for the primary use case.
