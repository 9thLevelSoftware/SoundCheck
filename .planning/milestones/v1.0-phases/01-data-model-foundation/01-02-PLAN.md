Write migration 009 that updates the database trigger functions to work with BOTH the old and new schema simultaneously. This is critical because:
- The existing `update_user_stats_on_checkin()` trigger directly references `checkins.band_id` and `checkins.venue_id`
- These columns will be removed in a future contract phase
- The trigger must gracefully handle BOTH old-style checkins (band_id + venue_id set, event_id null) AND new-style checkins (event_id set, old columns may be null)

**Replace `update_user_stats_on_checkin()` with a dual-path version:**

```sql
CREATE OR REPLACE FUNCTION update_user_stats_on_checkin()
RETURNS TRIGGER AS $$
DECLARE
  v_band_id UUID;
  v_venue_id UUID;
BEGIN
  IF TG_OP = 'INSERT' THEN
    -- Determine band_id and venue_id from either path
    IF NEW.event_id IS NOT NULL THEN
      -- New path: get venue from events table, band from event_lineup
      SELECT e.venue_id INTO v_venue_id FROM events e WHERE e.id = NEW.event_id;
      -- Use first headliner band for stats (or first band if no headliner)
      SELECT el.band_id INTO v_band_id
      FROM event_lineup el
      WHERE el.event_id = NEW.event_id
      ORDER BY el.is_headliner DESC, el.set_order ASC
      LIMIT 1;
    ELSE
      -- Legacy path: read directly from checkin columns
      v_band_id := NEW.band_id;
      v_venue_id := NEW.venue_id;
    END IF;

    -- Update total check-ins for user
    UPDATE users SET total_checkins = total_checkins + 1 WHERE id = NEW.user_id;

    -- Update unique bands count (union of both paths)
    UPDATE users SET unique_bands = (
      SELECT COUNT(DISTINCT band_id) FROM (
        SELECT band_id FROM checkins WHERE user_id = NEW.user_id AND band_id IS NOT NULL
        UNION
        SELECT el.band_id FROM checkins c
        JOIN event_lineup el ON c.event_id = el.event_id
        WHERE c.user_id = NEW.user_id AND c.event_id IS NOT NULL
      ) sub
    ) WHERE id = NEW.user_id;

    -- Update unique venues count (union of both paths)
    UPDATE users SET unique_venues = (
      SELECT COUNT(DISTINCT venue_id) FROM (
        SELECT venue_id FROM checkins WHERE user_id = NEW.user_id AND venue_id IS NOT NULL
        UNION
        SELECT e.venue_id FROM checkins c
        JOIN events e ON c.event_id = e.id
        WHERE c.user_id = NEW.user_id AND c.event_id IS NOT NULL
      ) sub
    ) WHERE id = NEW.user_id;

    -- Update band stats if we have a band
    IF v_band_id IS NOT NULL THEN
      UPDATE bands SET
        total_checkins = total_checkins + 1,
        unique_fans = (
          SELECT COUNT(DISTINCT user_id) FROM (
            SELECT user_id FROM checkins WHERE band_id = v_band_id
            UNION
            SELECT c.user_id FROM checkins c
            JOIN event_lineup el ON c.event_id = el.event_id
            WHERE el.band_id = v_band_id AND c.event_id IS NOT NULL
          ) sub
        ),
        average_rating = COALESCE(
          (SELECT AVG(rating) FROM (
            SELECT rating FROM checkins WHERE band_id = v_band_id AND rating IS NOT NULL
            UNION ALL
            SELECT cbr.rating FROM checkin_band_ratings cbr WHERE cbr.band_id = v_band_id
          ) sub),
          0
        )
      WHERE id = v_band_id;
    END IF;

    -- Update venue stats if we have a venue
    IF v_venue_id IS NOT NULL THEN
      UPDATE venues SET
        total_checkins = total_checkins + 1,
        unique_visitors = (
          SELECT COUNT(DISTINCT user_id) FROM (
            SELECT user_id FROM checkins WHERE venue_id = v_venue_id
            UNION
            SELECT c.user_id FROM checkins c
            JOIN events e ON c.event_id = e.id
            WHERE e.venue_id = v_venue_id AND c.event_id IS NOT NULL
          ) sub
        ),
        average_rating = COALESCE(
          (SELECT AVG(r) FROM (
            SELECT rating AS r FROM checkins WHERE venue_id = v_venue_id AND rating IS NOT NULL
            UNION ALL
            SELECT c.venue_rating AS r FROM checkins c
            JOIN events e ON c.event_id = e.id
            WHERE e.venue_id = v_venue_id AND c.venue_rating IS NOT NULL
          ) sub),
          0
        )
      WHERE id = v_venue_id;
    END IF;

    -- Update event total_checkins if applicable
    IF NEW.event_id IS NOT NULL THEN
      UPDATE events SET total_checkins = total_checkins + 1 WHERE id = NEW.event_id;
    END IF;
  END IF;

  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

**Migration 010: Migrate shows to events**

Step-by-step SQL operations (all in a single `pgm.sql()` per step for transaction safety):

1. Copy shows into events table. Map columns:
   - `id` -> `id` (preserve UUIDs for FK continuity with notifications.show_id)
   - `venue_id` -> `venue_id`
   - `show_date` -> `event_date`
   - `doors_time`, `start_time`, `end_time` -> same names
   - `ticket_url`, `ticket_price_min`, `ticket_price_max` -> same names
   - `is_sold_out`, `is_cancelled` -> same names
   - `description` -> `description`
   - `source` -> `'migrated'`
   - `is_verified` -> `true` (existing data is verified)
   - `created_at`, `updated_at` -> same
   - `event_name` -> NULL (shows didn't have names)
   - Use `ON CONFLICT DO NOTHING` for idempotency

2. Create event_lineup entries from shows (each show had exactly one band):
   ```sql
   INSERT INTO event_lineup (event_id, band_id, set_order, is_headliner)
   SELECT id, band_id, 0, true
   FROM shows
   ON CONFLICT DO NOTHING;
   ```

3. Migrate notifications.show_id to notifications.event_id:
   ```sql
   UPDATE notifications n
   SET event_id = n.show_id
   WHERE n.show_id IS NOT NULL AND n.event_id IS NULL;
   ```
   (This works because we preserved the same UUIDs from shows -> events)

The `down` function should be a SAFE no-op with a comment explaining that data migration rollback is not destructive because the expand phase preserved all original columns and tables.

**Migration 011: Backfill checkins data**

Step-by-step:

1. Match checkins to events by joining through venue_id + band_id + event_date:
   ```sql
   UPDATE checkins c
   SET event_id = e.id
   FROM events e
   JOIN event_lineup el ON e.id = el.event_id
   WHERE c.venue_id = e.venue_id
     AND c.band_id = el.band_id
     AND c.event_date = e.event_date
     AND c.event_id IS NULL;
   ```

2. For checkins that could not match (orphaned -- no corresponding show), create events on-the-fly:
   ```sql
   INSERT INTO events (venue_id, event_date, source, is_verified, created_at, updated_at)
   SELECT DISTINCT c.venue_id, c.event_date, 'migrated', false, MIN(c.created_at), MIN(c.created_at)
   FROM checkins c
   WHERE c.event_id IS NULL
     AND c.venue_id IS NOT NULL
     AND c.event_date IS NOT NULL
   GROUP BY c.venue_id, c.event_date
   ON CONFLICT DO NOTHING;
   ```

3. Create event_lineup entries for orphaned checkins:
   ```sql
   INSERT INTO event_lineup (event_id, band_id, set_order, is_headliner)
   SELECT DISTINCT e.id, c.band_id, 0, true
   FROM checkins c
   JOIN events e ON c.venue_id = e.venue_id AND c.event_date = e.event_date
   WHERE c.event_id IS NULL
     AND c.band_id IS NOT NULL
   ON CONFLICT DO NOTHING;
   ```

4. Re-run the backfill for newly created events:
   ```sql
   UPDATE checkins c
   SET event_id = e.id
   FROM events e
   WHERE c.venue_id = e.venue_id
     AND c.event_date = e.event_date
     AND c.event_id IS NULL;
   ```

5. Copy old single rating to venue_rating:
   ```sql
   UPDATE checkins
   SET venue_rating = rating
   WHERE venue_rating IS NULL AND rating IS NOT NULL;
   ```

6. Create band ratings from old single rating:
   ```sql
   INSERT INTO checkin_band_ratings (checkin_id, band_id, rating)
   SELECT c.id, c.band_id, c.rating
   FROM checkins c
   WHERE c.rating IS NOT NULL
     AND c.band_id IS NOT NULL
   ON CONFLICT DO NOTHING;
   ```

7. Copy comment to review_text:
   ```sql
   UPDATE checkins
   SET review_text = comment
   WHERE review_text IS NULL AND comment IS NOT NULL;
   ```

8. Copy photo_url to image_urls array:
   ```sql
   UPDATE checkins
   SET image_urls = ARRAY[photo_url]
   WHERE image_urls IS NULL AND photo_url IS NOT NULL;
   ```

The `down` function is a no-op (data preserved in original columns).
