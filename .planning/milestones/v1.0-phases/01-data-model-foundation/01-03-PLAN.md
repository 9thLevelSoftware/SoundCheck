**Rewrite EventService.ts:**

Replace the entire service to query `events` + `event_lineup` instead of `shows`. Key changes:

1. **createEvent()**: INSERT INTO events + INSERT INTO event_lineup (not shows). Accept an optional `lineup` array parameter for multi-band events. For backward compat with the existing `createEvent({venueId, bandId, eventDate})` signature, if a single `bandId` is provided, create a single lineup entry.

   First check for existing event: `SELECT FROM events e JOIN event_lineup el ON e.id = el.event_id WHERE e.venue_id = $1 AND el.band_id = $2 AND e.event_date = $3`. If exists, return it.

   If not, create event + lineup entry in a transaction (use `this.db.getClient()` for transaction):
   ```typescript
   const client = await this.db.getClient();
   try {
     await client.query('BEGIN');
     const eventResult = await client.query('INSERT INTO events ... RETURNING *', [...]);
     await client.query('INSERT INTO event_lineup ...', [...]);
     await client.query('COMMIT');
   } catch (err) {
     await client.query('ROLLBACK');
     throw err;
   } finally {
     client.release();
   }
   ```

2. **getEventById()**: Query events JOIN event_lineup JOIN bands JOIN venues. Return the event with its full lineup array (not just one band).

3. **getEventsByVenue()**: Query events + lineup where venue_id matches. Include checkin counts.

4. **getEventsByBand()**: Query events JOIN event_lineup where band_id matches.

5. **getUpcomingEvents()**: Query events where event_date >= CURRENT_DATE AND is_cancelled = FALSE.

6. **getTrendingEvents()**: Query events with checkin counts from last 30 days, ordered by count.

7. **deleteEvent()**: DELETE FROM events (CASCADE handles lineup entries).

8. **findOrCreateEvent(venueId, bandId, eventDate)**: New helper method. Looks up an event by venue+date, and if the band is in the lineup. If found, returns the event. If not, creates event + lineup entry. This is used by CheckinService for dual-write.

All queries should return `Event` type (not `Show`). The mapper function should be `mapDbEventToEvent()`.

IMPORTANT: The response shape must be BACKWARD COMPATIBLE with what EventController currently returns. The existing mobile app expects the shape that `mapDbShowToShow` produces. To maintain compat:
- Include a `bandId` field on the Event response pointing to the headliner band (first in lineup)
- Include `band` and `venue` objects in the same shape as before
- Include `showDate` as an alias for `eventDate`

This ensures the mobile app continues to work without changes.
