# v2.0 Requirements — Beta Launch

**Defined:** 2026-03-01
**Source:** Board of Directors Beta Readiness Assessment (5-0 APPROVE WITH CONDITIONS)
**Goal:** Close all blockers identified by the 5-member board (CA, CPO, CSO, COO, CXO) to ship a live beta to 50-500 users.

---

## Security & Infrastructure (9)

- [ ] BETA-01: Rotate all exposed secrets (DB password, JWT_SECRET, SetlistFM API key) and store exclusively in Railway environment variables — never in .env or git
- [ ] BETA-02: Set NODE_ENV=production in Railway — fixes CORS wildcard, stack trace leaks, query logging, and logger file rotation
- [ ] BETA-03: Configure all missing environment variables in Railway: REDIS_URL, TICKETMASTER_API_KEY, FIREBASE_SERVICE_ACCOUNT_JSON, CLOUDFLARE_ACCOUNT_ID, R2_ACCESS_KEY_ID, R2_SECRET_ACCESS_KEY, R2_BUCKET_NAME, REVENUECAT_WEBHOOK_AUTH, SENTRY_DSN, ENABLE_WEBSOCKET=true
- [ ] BETA-04: Add `app.set('trust proxy', 1)` before rate limiting middleware — Railway reverse proxy makes req.ip return proxy IP, breaking rate limits and audit logs
- [ ] BETA-05: Install gitleaks or detect-secrets pre-commit hook to prevent future secret commits
- [ ] BETA-06: Fix CI pipeline branch trigger — ci.yml targets main/develop but active branch is master
- [ ] BETA-07: Make auth rate limiter fail-closed (return 503) instead of fail-open when Redis is unavailable — brute-force protection must not degrade silently
- [ ] BETA-08: Replace SOCIAL_AUTH_NO_PASSWORD sentinel (`$SOCIAL_AUTH$`) with bcrypt hash of cryptographically random value — current plaintext sentinel leaks account type if DB is exfiltrated
- [ ] BETA-09: Delete .bak/.backup files from backend/src/services/ and add *.bak, *.backup to .gitignore

## Mobile UX & Accessibility (6)

- [ ] BETA-10: Wire onboarding into router — check hasSeenOnboarding in app_router.dart redirect function, route first-time users to onboarding carousel + genre picker before login
- [ ] BETA-11: Wire Terms of Service and Privacy Policy URLs in pro_feature_screen.dart — currently empty onPressed handlers, which is an App Store rejection blocker for subscription screens
- [ ] BETA-12: Ship dark-only theme for beta — disable Light/System options in settings theme picker. 686 hardcoded AppTheme.*Dark color references across 43 files break light mode entirely.
- [ ] BETA-13: Add actionable empty states to feed — Friends tab needs "Find Friends" or "Explore" button, Happening Now tab needs copy that makes sense for users with zero friends. Use existing EmptyStateWidget pattern.
- [ ] BETA-14: Add Semantics labels to bottom navigation bar — check-in button and nav items have no screen reader labels. Also add tooltips to all IconButtons across the app.
- [ ] BETA-15: Fix owner response submit button text color — white on voltLime has poor contrast; should be AppTheme.backgroundDark on voltLime matching the rest of the app

## Backend Data Integrity & Logging (4)

- [ ] BETA-16: Add block filter to WrappedService.getFriendOverlap() — blocked user activity currently leaks into Wrapped stats. Use existing BlockService.getBlockFilterSQL() pattern.
- [ ] BETA-17: Fix VenueService semantic mismatch — total_reviews DB column mapped to totalCheckins in API response. Rename or document the mapping correctly.
- [ ] BETA-18: Add idempotency guard to ReviewService.respondToReview — currently allows silent overwrite of owner responses. Add WHERE owner_response IS NULL or return 409 if response exists.
- [ ] BETA-19: Replace console.log/error/warn calls with structured winston logger — 455 console statements across 69 backend source files bypass the existing logger utility

## Operational Readiness (5)

- [ ] BETA-20: Configure Sentry DSN in Railway and verify error capture with a test error — currently disabled despite full integration code existing
- [ ] BETA-21: Set up uptime monitoring on /health endpoint — UptimeRobot or Better Uptime free tier with email/SMS alerts on downtime
- [ ] BETA-22: Write deployment runbook — pre-deploy checklist, deploy steps, post-deploy smoke test procedure, rollback procedure using Railway dashboard
- [ ] BETA-23: Create critical-path smoke tests — 5-10 tests hitting live API to verify core flows (register, login, check-in, feed load, review submit) after each deployment
- [ ] BETA-24: Create staging environment on Railway — clone service with separate database for pre-deployment validation

## Social Graph & Beta Onramp (4)

- [ ] BETA-25: Add friend discovery / suggested users screen — users need a way to find and follow other beta testers. Even basic username search with a beta user directory.
- [ ] BETA-26: Elevate share CTA on celebration screen — move share section above badge progress or make it the primary action button. The viral loop depends on post-check-in sharing.
- [ ] BETA-27: Seed beta with demo content — create sample accounts, check-ins, and events so new users see activity in the global feed on day one
- [ ] BETA-28: Update purchases_flutter to non-deprecated API — replace Purchases.purchasePackage() with Purchases.purchase(PurchaseParams) before RevenueCat removes it
